## Domain structure

Each domain owns all its artefacts: `router.py`, `schemas.py`, `models.py`, `dependencies.py`, `service.py`, `repository.py`, `constants.py`, `exceptions.py`. Cross-domain imports must be explicit (`from app.auth import constants as auth_constants`).

This project uses top-level `app/`; it is equivalent to the `src/` directory used in `zhanymkanov/fastapi-best-practices`.

## Response envelope

All endpoints return `ApiResponse[T]`: `{"code": 200, "message": "success", "data": {...}}`. Use `ApiResponse.ok(data)` in routers. Errors: `{"code": 4xx, "message": "...", "data": null}`.

This consistency is intentional even though FastAPI validates against `response_model` after the router builds the response object. For extremely hot endpoints, measure first before bypassing the envelope.

## Router handlers

- Every route decorator must include `summary`, `description`, and `response_model`.
- No docstrings inside handler functions — text goes in the decorator.
- `response_model` must be explicit; document non-200 codes with `responses=`.

## Schemas

- All schemas inherit from `app.core.schemas.CustomModel` (`populate_by_name=True`, `from_attributes=True`).
- Every `Field` must have `description` and `examples`. Input schemas add constraints (`min_length`, `max_length`, `ge`, `le`). Output schemas (e.g. `UserRead`) skip constraints.
- Use `EmailStr` for email fields. Use `@field_validator` for field-level business rules — `ValueError` becomes `ValidationError` automatically.

## Dependencies

- Resource-loading deps raise `HTTPException` directly. Domain services raise `DomainError` subclasses.
- Chain dependencies freely — FastAPI caches per-request, no overhead.
- Auth guard: `CurrentUser = Annotated[User, Depends(get_current_active_user)]`.
- Public endpoints must be intentional. Any endpoint that reads user, tenant, billing, or private domain data should default to `CurrentUser` or a narrower guard.

## Async / sync

- `async def` only for non-blocking `await` I/O. Blocking inside async freezes the event loop.
- `def` (sync) for blocking libraries — FastAPI runs it in a threadpool.
- Sync SDK inside async route → `await run_in_threadpool(sync_fn, args)`.

## Database

- New ORM models must be imported in both `app/db/session.py` and `alembic/env.py`.
- Table names: `lower_snake_case`, singular, domain-prefixed. `_at` for datetime columns, `_date` for date columns.
- Migrations: `make revision m="describe change"`, review autogenerated output, must be reversible.

## Settings

All env vars use `APP_` prefix. Access via `get_settings()` (LRU-cached singleton). Docs are hidden (`openapi_url=None`) outside `development` / `test` environments.
