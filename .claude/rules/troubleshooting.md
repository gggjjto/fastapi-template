## Test container errors

**Symptom:** `Connection refused` to PostgreSQL (5433) or Redis (6380) when running tests.

```bash
make test-up        # start containers
docker ps           # verify they are running
make test           # retry
```

If containers start but tests still fail: check `.env` for `APP_DATABASE_URL` and `APP_REDIS_URL` — tests use `os.environ.setdefault` in `conftest.py`, so a wrong env file value can override the test defaults.

## Migration failures

**Symptom:** `alembic upgrade head` fails or autogenerated migration is empty.

1. Verify new ORM models are imported in **both** `app/db/session.py` and `alembic/env.py`. Alembic only sees models that are imported before `target_metadata` is read.
2. If the migration is empty: the model import is missing. Add it, then re-run `make revision m="..."`.
3. If `alembic upgrade head` fails with a constraint error: check for a conflicting partial index or duplicate key — review the generated file before applying.
4. Never hand-write a new revision file. Edit the autogenerated one only for steps autogenerate cannot produce (data backfills, partial indexes).

## Mypy errors

**Symptom:** Cryptic type errors after a change.

Common causes and fixes:

| Error pattern | Cause | Fix |
|---|---|---|
| `Missing return statement` | `async def` missing `return` | Add explicit `return` or adjust return type |
| `Incompatible types in assignment` on SQLAlchemy column | `Column[T]` vs plain `T` | Use `Mapped[T]` with `mapped_column()` |
| `Module has no attribute` on an optional service | Optional service not imported | Guard with `TYPE_CHECKING` import |
| `Argument 1 to ... has incompatible type` on Pydantic model | `model.field` is `T | None`, caller expects `T` | Assert non-None or adjust the type |

Do not suppress with `# type: ignore` unless the root cause is a known external library type mismatch. Always add a comment explaining why.

## Ruff lint / format failures

**Auto-fix first, then check:**

```bash
uv run ruff check --fix .
uv run ruff format .
uv run ruff check .          # should be clean
uv run ruff format --check . # should be clean
```

If `ruff check` still fails after `--fix`: the remaining errors require manual fixes (usually `E501` long lines or `ERA001` commented-out code).

## Redis / arq not available

**Symptom:** `ArqPool` or `RedisClient` dependency raises at startup.

Both are opt-in. If `APP_REDIS_URL` is not set, the app starts without them but any route that injects `ArqPool` or `RedisClient` will fail at request time. Set `APP_REDIS_URL=redis://localhost:6380/0` in `.env` and restart.

---

## Test isolation failures

**Symptom:** One test affects another (unexpected data present, state leaking).

Tests rely on `_reset_state` autouse fixture in `conftest.py` which calls `reset_db()` + `FLUSHDB` before every test. If a test creates state outside these mechanisms (e.g. a file on disk, an env var mutation), clean it up in the test itself with a `try/finally` or `yield` fixture.
