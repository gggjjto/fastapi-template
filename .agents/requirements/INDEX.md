# Requirements Directory Index

The `.agents/requirements.md` file remains the short, active requirements ledger.
When a single domain grows, each domain can be moved into a dedicated file below.

## Active requirement domains

- `agent-workflow.md` — AI workflow, requirements lifecycle, and change governance.

## Update rule

- Keep `requirements.md` concise and authoritative for active decisions.
- Keep deeper rationale and process details in this directory.
- When `requirements.md` has any section with more than ~5 active items, split that section into a domain file under `.agents/requirements/` and link back from `requirements.md`.
